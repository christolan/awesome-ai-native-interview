## Question

在 AI Native 工程中，你接触过支持 reasoning / thinking 的模型（如 DeepSeek-R1、OpenAI o1、Claude Extended Thinking）。请从模型运作机制层面解释：

1. **thinking tokens 是什么？** 它们和普通 output tokens 在机制上有什么本质区别？
2. **为什么让模型"先想后答"能提升复杂推理的准确性？** 请从自回归生成的特性来解释，而不是说"因为 CoT 有用"。
3. **从工程角度，处理 reasoning 模型的流式响应时，和普通模型有什么不同？**

## Why this matters

Reasoning 模型是 AI Native 工程的前沿话题，考察候选人对 LLM 生成机制的深层理解，而非 API 表面用法。

## Expected points

- 理解 thinking tokens 和 output tokens 在生成机制上完全一样，都是自回归 next token prediction
- 理解"先想后答"提升准确性的原因：把压缩推理变成显式推理，每个中间 token 都在缩小下一步不确定性
- 理解双状态机：reasoning 和 answering 两个阶段，thinking tokens 默认不展示
