## AI Explanation

用户未作答（本轮用于验证 AI 维护能力），由 AI 直接讲解。

**1. thinking tokens 是什么？**

thinking tokens 和普通 output tokens 在生成机制上完全一样——都是自回归 next token prediction，模型本身不区分"思考"和"回答"。区别在于语义边界：thinking tokens 是模型在正式回答前生成的内部推理链，默认对用户不可见。

**2. 为什么"先想后答"能提升准确性？**

标准自回归生成中，模型必须在一个 forward pass 中直接从问题跳到答案。复杂推理本质上需要中间步骤——传统模型只能把这些步骤压缩到隐层状态中。

Reasoning 模型把压缩推理变成显式推理：每个 thinking token 都是中间推理步骤，作为后续 token 的条件。等效于将 P(answer|question) 分解为 P(answer|thought_n) × ... × P(thought_1|question)，每一步都在缩小不确定性。

**3. 工程上处理 reasoning 模型流式响应的不同**

- SSE delta 同时携带 `reasoning_content` 和 `content`，需要双缓冲区
- 通过边界信号（如 `<｜end▁of▁thinking｜>` 和 ` ` token）判断当前阶段是 thinking 还是 answering
- UI 分层渲染：thinking tokens 到可折叠思考面板，answering tokens 到主对话区
- 注意 thought 和 answer 之间不能直接 markdown 拼接，未闭合代码块会污染渲染

## Improved Answer

Thinking tokens 和 output tokens 在生成机制上完全相同——都是自回归 next token prediction。区别是语义层面：thinking tokens 构成推理链，answering tokens 构成最终回复。

"先想后答"通过显式化推理链，将 P(answer|question) 分解为更短的链式条件概率，逐步收窄搜索空间。

工程上需双缓冲区 + 边界信号解析 + UI 分层渲染，核心是理解两种 content 的区分和阶段切换时机。
