# 2026-05-01 面试记录

## 题目

Beam Search 的原理是什么？它和 greedy decoding 有什么区别？相比随机采样有什么优缺点？为什么现代对话模型普遍不用它？

## 作答过程

用户未作答，由 AI 直接讲解。随后用户对 Beam Search 展开了深度追问：

1. 每条路径具体怎么计算概率？
2. 多条分支的下一个 token 是怎么得到的？
3. 每步是不是把词表所有词都和已有句子拼在一起计算概率？
4. 大模型怎么知道什么时候停下来？

## 核心结论

- Beam Search 每步保留 Top-B 条候选路径，每步累积 log 概率得分，最终 EOS 后取 Top-1 输出
- 模型一次 forward pass 输出整个词表的概率分布，Beam Search 只是对这个分布的不同利用方式
- beam size=1 退化为 greedy decoding
- 停止条件：遇到 `<EOS>` 或达到 `max_tokens`
- 对话模型不用 Beam Search：缺乏多样性、倾向生成安全但无聊的句子、开放域没有标准答案
