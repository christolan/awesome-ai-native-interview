## Question

Beam Search 的原理是什么？它和 greedy decoding 有什么区别？相比随机采样有什么优缺点？为什么现代对话模型普遍不用它？

## Why this matters

理解 Beam Search 是深入 LLM 解码策略的基础。它揭示了确定性搜索和随机采样之间的核心张力，以及为什么不同的 AI 应用场景（翻译 vs 对话）会选择截然不同的解码方案。这道题考察候选人对自回归生成底层逻辑的把握，而非 API 调参技巧。

## Expected points

- 理解 Beam Search 的核心思想：每步保留 Top-B 条候选路径，累积 log 概率得分
- 能清晰区分 greedy decoding（beam size=1）和标准 Beam Search
- 理解 beam search 与随机采样在多样性和确定性维度上的对立
- 能解释为什么对话模型不用 Beam Search：缺乏多样性、倾向安全但无聊的输出
- 知道停止条件：遇到 `<EOS>` 或达到 `max_tokens`

## AI Explanation

用户未作答，由 AI 直接讲解。

Beam Search 是自回归解码策略的一种。模型每步 forward pass 输出整个词表的概率分布（logits → softmax → 概率向量）。不同解码策略的区别在于：**如何从这个概率分布中选出下一个 token**。

**Greedy Decoding**：每步只选概率最高的 1 个 token。这是 Beam Search 在 beam size=1 时的退化形式。

**Beam Search**：每步保留 Top-B 条候选路径（B 即 beam size），每条路径累积 log 概率得分。下一步对所有 B 条路径的每种可能延伸分别计算得分，再从 B × |vocab| 个候选中保留新的 Top-B。最终遇到 `<EOS>` 后取累积得分最高的 1 条完整路径。

**和随机采样的对比**：
- Beam Search 是确定性的，始终选最高累积得分的路径 → 输出质量稳定，但多样性低
- 随机采样（temperature + top_p / top_k）从概率分布中取样 → 输出多样，但可能不稳定
- Beam Search 适合需要"最佳输出"的场景（机器翻译、OCR），随机采样适合需要"自然多样"的场景（对话、创意写作）

**为什么对话模型不用它**：对话本质上没有标准答案。Beam Search 倾向于生成语法正确但内容无聊的"安全回答"，缺乏人类对话的多样性和意外性。而且 beam size 越大，推理延时越高——这对实时对话是不可接受的。

随后用户对 Beam Search 展开了深度追问：

1. **每条路径具体怎么计算概率？** — 每步对每条路径的最后一个 token 输出取 log 概率，累加到该路径的历史累积得分。
2. **多条分支的下一个 token 是怎么得到的？** — 一次 forward pass 就输出整个词表的概率分布，同一个分布同时用于所有 B 条路径的下一个 token 选择。
3. **每步是不是把词表所有词都和已有句子拼在一起计算概率？** — 不需要逐个拼接。一次 forward pass 输入 B 条路径的 KV Cache，输出 B × |vocab| 的概率矩阵。
4. **大模型怎么知道什么时候停下来？** — 当某条路径的 top-1 token 是 `<EOS>` 时，该路径终止并记录最终得分；其他路径继续直到也遇到 `<EOS>` 或达到 `max_tokens`。

## Improved Answer

Beam Search 是一种确定性解码策略，核心是**每步保留 Top-B 条候选路径**，而非只保留 1 个最优（greedy）或完全随机采样。

每次 forward pass，模型基于当前的 B 条路径各自输出整个词表的概率分布。对每条路径的每个可能下一个 token 计算累积得分（log 概率之和），从 B × |vocab| 个候选中再选出新的 Top-B。遇到 `<EOS>` 的路径被标记为完成，最终取累积得分最高的一条。

**核心性质**：
- beam size=1 退化为 greedy decoding
- 一次 forward pass 输出整个词表分布，不需要逐 token 拼接
- 停止条件：`<EOS>` 或 `max_tokens`

**优缺点**：
- 优点：输出质量稳定、数学上保证局部最优
- 缺点：缺乏多样性、长文本中易陷入重复循环、推理延迟随 beam size 线性增长

**为什么对话模型不用**：对话没有标准答案，Beam Search 的"最优"往往是最无聊的答案。随机采样（temperature + top_p）更符合人类对话的多样性需求。
