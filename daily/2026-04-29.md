# 2026-04-29

## Question

Transformer 的注意力机制和 FFN 是如何协作完成"预测下一个 token"这件事的？

## Why this matters

理解 Transformer 内部的计算流程，是真正理解 LLM 工作原理的基础。这不只是面试题，也是理解 KV Cache、上下文窗口成本、temperature 参数、模型幻觉等工程问题的前提。

## Expected points

- 注意力机制的 QKV 计算过程，以及最终向量的含义
- FFN 的结构（升维 → 激活 → 降维）和作用
- Block 堆叠的含义：每层有独立的 QKV 权重，向量逐层被加工
- KV Cache 的结构：按层存储，层数越多缓存越大
- 从最后一层输出向量到 Logits 再到概率分布的过程
- 采样策略：贪心、temperature、top-p 的区别
- 为什么 LLM 每次回答不一样

## Your Answer

> 本次以深度问答形式展开，用户通过追问逐步建立起完整的心智模型。

**注意力机制输出的向量是什么？**

计算下一个 token 时，先算最后一个 token 的 QKV，用它的 Q 和所有历史 token 的 K 计算权重，再把权重和对应的 V 加权求和，得到一个向量。这个向量是"当前 token 在当前上下文中的语义表示"。

**FFN 做了什么？**

注意力层的加权求和是线性操作，FFN 在此基础上做非线性变换：先把向量升维（如 768 → 3072），经过 ReLU 激活函数将负值归零，再降维回原始维度（3072 → 768）。升维给了模型足够空间学习复杂特征，ReLU 引入非线性（否则两次线性变换等于一次）。

**堆叠 N 层的含义？**

不是 FFN 重复 N 次，而是"注意力 + FFN"这整个 Block 重复 N 次。每一层有独立的 QKV 权重矩阵，输入是上一层的输出向量，因此每层提取的是不同抽象层次的语义。

**KV Cache 的规模？**

KV Cache 按层存储，每层保存自己的 K 和 V。层数越多、序列越长，缓存越大。这是 LLM 推理的核心显存瓶颈之一。业界用 MQA、GQA、KV 量化等方式压缩它。

**最终如何预测 token？**

最后一层输出的向量，乘以输出矩阵（维度 = 词表大小），得到 Logits（原始打分），再经 Softmax 转为概率分布，最后按采样策略选出下一个 token。每次预测都对整个词表打分。

**为什么每次回答不一样？**

Logits 是确定的，但采样有随机性。temperature=0 时完全确定，temperature 越高越随机。top-p 则限制候选词范围再随机采样。

## Improved Answer

Transformer 的推理流程大致分为四个阶段：

**1. 注意力层：聚合上下文**
每个 token 通过线性变换生成 Q、K、V 三个向量。当前 token 的 Q 与所有历史 token 的 K 做相似度计算，得到注意力权重，再对所有 token 的 V 加权求和，输出一个"上下文感知的语义向量"。历史 token 的 K、V 会被缓存（KV Cache），避免重复计算。

**2. FFN：非线性加工**
注意力输出的向量进入 FFN：先升维（×4）再用 ReLU 激活，最后降维回原始维度。FFN 的作用是引入非线性表达能力，同时被认为存储了模型的大量事实性知识。

**3. 多层堆叠**
以上两步合为一个 Block，GPT 类模型会堆叠几十到上百个 Block。每层有独立权重，浅层学语法，深层学抽象语义和推理。KV Cache 也是按层存储，层数 × 序列长度决定缓存大小，是长上下文推理成本高的核心原因。

**4. 解码：从向量到 Token**
最后一层输出的向量，乘以词表大小的输出矩阵，得到 Logits（原始分数），经 Softmax 转为概率分布，再通过采样策略选出下一个 token：
- `temperature=0`：贪心，始终选最高概率，结果可复现
- `temperature>0`：按概率随机采样，越高越随机
- `top-p`：只保留累计概率达到 p 的候选词再随机采样

每次预测都对整个词表打分，这也是词表大小影响推理成本的原因之一。
